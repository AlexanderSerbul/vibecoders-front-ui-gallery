# Как выложить свой сайт в интернет (Вайбкод)

Этот сайт можно опубликовать на платформе **Битрикс24 Вайбкод** — без программирования и без знания серверов. Ниже два способа: совсем простой (попросить ИИ-агента) и «одной командой».

## Что нужно один раз

- **Ключ Вайбкода** — заведи его в личном кабинете Вайбкода (раздел «Ключи API»). Это строка вида `vibe_api_…`. Никому не показывай и в код не клади.
- На портале должна быть активна подписка **BitrixGPT + Маркетплейс** — без неё платформа не выдаёт серверы.
- **Node.js** — обязателен: им собирается сайт (`npm run build`). Скачать: [nodejs.org](https://nodejs.org).
- **Если ты на Windows** — установи [Git Bash](https://git-scm.com/download/win) (идёт в составе Git for Windows): именно в нём запускаются команды деплоя. На macOS и Linux он уже есть.
- **Python** — для самого сайта не нужен (он на JavaScript), но пригодится при разработке: ИИ-ассистент часто пишет на нём небольшие вспомогательные скрипты и быстро проверяет идеи. Скачать: [python.org/downloads](https://www.python.org/downloads/).

## Способ 1 — просто попроси ИИ-агента (рекомендую)

Открой своего ИИ-агента (Claude Code, Cursor и т.п.) в этом проекте и напиши примерно так:

> Задеплой этот сайт на Вайбкод. Мой ключ: `vibe_api_…`
> Скрипт и инструкция — в папке `deploy/`.

Агент сам всё сделает: соберёт сайт, поднимет сервер, зальёт и пришлёт ссылку. Тебе останется открыть адрес. Это в духе всего сайта — **за тебя работает агент**, ты только говоришь, что нужно.

## Способ 2 — одной командой (если дружишь с терминалом)

На **Windows** запускай команду в **Git Bash** (см. раздел «Что нужно один раз»), а не в обычном PowerShell.

```bash
VIBE_KEY=vibe_api_твой_ключ  bash deploy/deploy.sh
```

В конце скрипт напечатает адрес вида `https://app-xxxxx.vibecode.bitrix24.tech`.

Запускай **ту же команду** каждый раз, когда поправил сайт, — он обновится. Сервер при этом переиспользуется (запоминается в файле `deploy/.vibe-server`), заново не создаётся.

## Что получится

- Сайт **публичный** — открывается у любого по ссылке, без входа в Битрикс24.
- Сайт **не засыпает** — работает круглосуточно, отклик всегда мгновенный.
- Это **платно**: сервер `bc-small` ≈ 1200 ₽/мес списывается с баланса Вайбкода.

## Если что-то пошло не так

- **«Сервер не создался» / ошибка 402** — пополни баланс Вайбкода или проверь подписку BitrixGPT + Маркетплейс.
- **Хочешь дешевле** — открой `deploy/deploy.sh` и поменяй `PLAN` (например, на `bc-agent`, ~300 ₽/мес), или сделай, чтобы сервер засыпал без посетителей: в самом низу скрипта замени `"sleepAfterMinutes":null` на `"sleepAfterMinutes":60`.

### Под капотом (для любопытных и для ИИ-агента)

`deploy/deploy.sh` делает 5 шагов: `npm run build` → пакует `dist/` вместе с `deploy/nginx.conf` → создаёт (или переиспользует) сервер → заливает архив с рантаймом `static` и командой запуска `nginx -c /opt/app/nginx.conf` → ставит публичный доступ и отключает авто-сон.

Два места, где легко споткнуться (проверено на граблях):

1. Приложение **обязано слушать порт 3000** — через него идёт туннель платформы, на него же стучится проверка после деплоя.
2. Команда запуска (`start`) **не должна содержать кавычек** — платформа их ломает (`Unterminated quoted string`). Поэтому флаг `daemon off;` лежит **внутри** `deploy/nginx.conf`, а в команде остаётся голое `nginx -c /opt/app/nginx.conf`.

Сам `deploy/nginx.conf` — это 9 строк: раздать `/opt/app` на порту 3000 и отдавать `index.html` на неизвестных путях (`try_files`), чтобы работали прямые ссылки вроде `/components/dialog`.

Файл `deploy/.vibe-server` (появляется после первого запуска) хранит id сервера — он локальный, в гит его класть не нужно.
